# PR Clipboard - Pull Request文書生成とクリップボードコピー

このスキルは、現在のgitブランチの変更内容を分析し、PR（Pull Request）文書をMarkdown形式で生成してクリップボードにコピーします。

## あなたの役割

以下の手順でPR文書を生成し、クリップボードにコピーしてください：

1. **ブランチと変更内容の確認**
   - 現在のブランチ名を取得
   - ベースブランチ（master/main）との差分を確認
   - 変更されたファイルのリストを取得
   - コミット履歴を確認

2. **変更内容の分析**
   - 変更されたファイルの内容を確認
   - 変更の目的と影響範囲を理解
   - 主な変更点を整理

3. **PR文書の生成**
   以下の構造でMarkdown文書を生成：

   ```markdown
   # [タイトル：変更内容を簡潔に表現]

   # 概要

   この変更の目的と背景を説明

   ## 変更内容

   主な変更点を箇条書きで概要がわかるように記載。詳細はコードレビューでわかるように

   ## テスト

   - [ ] 実施すべきテスト項目

   ## 注意事項

   レビュー時やマージ後の注意点（あれば）
   ```

4. **クリップボードへのコピー**
   - 生成したMarkdownを`pbcopy`コマンドでクリップボードにコピー
   - 成功を確認してユーザーに通知

## 重要な注意事項

- PR文書は日本語で生成してください
- 技術的に正確で、レビュワーが理解しやすい内容にしてください
- 生成したPR文書の内容をユーザーに表示してから、クリップボードにコピーしてください

## 実行方法

ユーザーが `/pr-clipboard` コマンドを実行すると、このスキルが起動します。
